# TDDD: TODO Driven Development (2026 Edition)

## 概要

**TDDD（TODO Driven Development）** は、  
TODO を中心に開発プロセス全体を回す、生成AI時代向けの開発手法である。

仕様・設計・実装・レビューはすべて **TODO に集約** され、  
開発とは「TODOを作り、実行し、更新し続けるループ」であると定義する。

本手法は、同一ツール内でも
**設計モード（判断）** と **実装モード（実行）** を意識的に切り替えることを前提としているため、
**年次版** として記録される。

---

## 背景

従来の開発手法は以下を前提としていた。

- 人間が仕様を理解し
- 人間が設計を分解し
- 人間がコードを書く

生成AIの普及により、  
**「実装を書くこと」自体は最も安価で不確実性の低い作業** になった。

一方で、以下は依然として人間側の責務である。

- 何を作るか
- どこまで作るか
- どこで止めるか
- 期待結果は何か

TDDD は、この **判断と実装の分離** を前提に設計されている。

---

## 基本思想

- 開発を駆動するのはコードではなく **TODO**
- TODO は「命令」ではなく「判断の記録」
- 実装は TODO の忠実な実行結果にすぎない
- 問題が起きた場合、修正対象はコードではなく **TODO**

---

## TDDD の基本ループ

TDDD は以下の 4 つのループで構成される。

Claude Code や Copilot Agent など、1つのツールが複数の役割を担う場合でも、
TDDD ではフェーズを意識的に分ける。

### Loop A：TODO 生成（判断フェーズ＋人間）

**目的**  
- 仕様理解
- 境界の明確化
- 作業単位への分割

**主な作業**
- 会話・資料・既存コードを入力として与える
- AI が暴れてよい範囲（境界）を明示する
- 境界内だけを対象とした TODO を作成する
- 必要であれば YAML / スキーマを生成する

**成果物**
- TODO.md
- （必要に応じて）schema.yaml / boundaries.md

---

### Loop B：TODO 実行（実行フェーズ）

**目的**  
- TODO に基づく機械的な実装

**主な作業**
- TODO フォルダを入力として与える
- 特定言語・特定制約下でコードを生成させる
- 挙動が曖昧な部分は実装せず TODO に差し戻す

**成果物**
- 実装コード
- 最低限のテストコード

---

### Loop C：動作確認（人間）

**目的**  
- 現実との乖離を確認する
- 責任を人間側に戻す

**主な作業**
- 実行・動作確認
- 期待と異なる挙動の観測
- 再現手順の整理

**成果物**
- 観測ログ
- 再現手順
- 新しい TODO

---

### Loop D：レビューと再TODO化（判断フェーズ）

**目的**  
- 学習の固定
- 次ループの精度向上

**主な作業**
- 実装結果を判断フェーズのAI操作でレビューする
- フィードバックを TODO に変換する
- 設計上の判断を短く記録する

**成果物**
- 更新された TODO
- 判断記録（任意）

---

## TDDD と他手法の関係

- TDDD は TDD（Test Driven Development）の代替ではない
- テストは TODO の一部として扱われる
- BDD / SpecDD と異なり、仕様書を主成果物としない

---

## 年次性について

TDDD は **ツール構成に強く依存** する。

- 同一ツール内でも、設計モード（判断）と実装モード（実行）を切り替える前提
- IDE内AIが両機能を統合していても
  → Loop A と Loop B は意識的に分けて扱う

そのため、本ドキュメントは **2026 Edition** として位置づける。

---

## 将来の拡張

- 境界設計を Loop A に組み込む高度化
- 組織的判断・責任設計との統合
- AI ガバナンスや設計原則の参照レイヤ追加

これらは TDDD の外部原則として扱い、  
TDDD 自体は **「TODOで回す」最小構造** を維持する。

---

## まとめ

TDDD は方法論ではなく **運転方法** である。

何を作るかではなく、  
**何を TODO として残すか** が開発の質を決める。

コードは消えても、  
TODO に記録された判断は次の実装に引き継がれる。
